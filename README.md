# 二阶魔方教学工具

## 项目概述

这是一个使用 Godot 引擎开发的二阶魔方教学工具，旨在通过可视化和数据建模的方式，帮助孩子们理解魔方的工作原理和建模概念。

## 工程架构

项目采用了清晰的三层架构：

1. **CubePiece**（角块基类）
   - 负责：显示、颜色、自身状态
   - 不负责：旋转逻辑

2. **Cube2x2**（魔方总控制器）
   - 负责：创建所有块、管理旋转、层转动、坐标更新
   - 旋转逻辑全部写在这里

3. **UIController**（输入控制器）
   - 负责：处理用户输入、显示魔方状态

### 2. 实时状态显示

界面上会实时显示当前魔方的状态，以数组（8个数字）的形式呈现。每次旋转后，状态会自动更新。

### 3. 教学模式

#### 数据、动作与模型的对应关系

- **数据**：数组（8个数字）表示的魔方状态
- **动作**：用户通过键盘输入进行的旋转操作
- **模型**：可视化的3D魔方模型

#### 教学步骤建议

1. **基础概念介绍**：
   - 介绍魔方的基本结构（8个角块）
   - 解释空间位置的概念
   - 解释朝向编码的概念
   - 打印编码表，包含8个位置的读取顺序。8个角在不同位置颜色对应的朝向编码。

2. **数据与模型对应**：
   - 展示初始状态下的数组（8个数字）
   - 解读数组中每个数字的含义，与实物对应。
   - 解释颜色如何映射到数字

3. **动作与数据对应**：
   - 执行一次旋转操作
   - 观察状态数字的变化
   - 分析哪些数字发生了变化，为什么

4. **模型与动作对应**：
   - 观察旋转时魔方的视觉变化
   - 解释旋转轴和方向
   - 理解层旋转的概念

5. **综合练习**：
   - 让孩子尝试不同的旋转组合
   - 观察状态数字的变化
   - 尝试根据状态数字预测魔方的外观

## 操作说明

- **U**：上层顺时针旋转，shift+U为逆时针旋转
- **D**：下层顺时针旋转，shift+D为逆时针旋转
- **L**：左层顺时针旋转，shift+L为逆时针旋转
- **R**：右层顺时针旋转，shift+R为逆时针旋转
- **F**：前层顺时针旋转，shift+F为逆时针旋转
- **B**：后层顺时针旋转，shift+B为逆时针旋转
- **空格**：重置魔方到初始状态，需要输入"reset"确认，防止误操作。

## 技术实现

- **CubePiece.gd**：角块基类，管理颜色和显示  
  每个角块是一个 `Node3D`，包含 1 个立方体网格（`CubeBody`）和 6 个平面贴纸网格（`Face_X+/-`、`Face_Y+/-`、`Face_Z+/-`）。  [朝向定义]呼应关系 Face_X+对应R,Face_X-对应L,Face_Y+对应U,Face_Y-对应D,Face_Z+对应F,Face_Z-对应B。
  代码中只在“外露的 3 个方向”上给贴纸上色,可以参考[8个位置读取顺序]里面的方向，那些方向是能看到的面，其他3个面是隐藏的。

- **Cube2x2.gd**：魔方总控制器，实现旋转逻辑和状态计算  
  内部使用：
  - `piece_positions`：长度为 8 的数组，约定 8 个角块的逻辑坐标顺序，坐标为 `(+/-0.5, +/-0.5, +/-0.5)`，这样 8 个角块的中心点刚好拼成一个 2×2×2 的正方体；

  旋转时，控制器会：
  - 先根据逻辑坐标筛选出某一层的 4 个角块（例如右层 `x>0`）；

## 设计思路与架构优化

### 设计原则

项目遵循以下软件工程原则：

1. **单一职责原则**：每个模块只负责一项核心功能
2. **职责分离**：明确划分各模块的职责，避免逻辑混乱
3. **封装性**：隐藏内部实现细节，提供清晰的接口
4. **可扩展性**：设计灵活，便于后续功能扩展
5. **性能优化**：针对关键操作进行性能改进

### 架构决策

1. **旋转逻辑集中管理**：所有旋转逻辑都集中在 `Cube2x2` 控制器中，而不是分散在各个角块中，这样可以更好地管理层旋转的复杂性

2. **数据结构设计**：
   - 使用 `pos_to_piece` 字典实现逻辑坐标到块实例的快速映射
   - 采用固定的 `piece_positions` 数组定义角块顺序，确保状态计算的一致性

3. **状态计算优化**：
   - 通过映射快速查找块，将状态计算的时间复杂度从 O(n²) 降低到 O(n)
   - 只在旋转操作后更新状态，避免不必要的计算

4. **代码组织**：
   - 清晰的变量和方法命名
   - 详细的注释说明
   - 模块化的代码结构

### 优化措施

1. **性能优化**：
   - 添加 `pos_to_piece` 映射，提高块查找效率
   - 移除不必要的 `_process` 方法，减少性能消耗

2. **代码质量优化**：
   - 移除重复代码，使用已定义的常量和数组
   - 保持代码风格一致，提高可读性
   - 添加详细注释，提高可维护性

3. **可维护性优化**：
   - 确保映射一致性，在旋转和重置操作后正确更新 `pos_to_piece` 映射
   - 保持职责分离，确保各模块功能清晰
   - 遵循面向对象原则，保持良好的封装性

4. **可扩展性优化**：
   - 模块化的设计便于添加新功能
   - 清晰的接口设计便于与其他系统集成
   - 灵活的数据结构便于后续扩展

## 总结

这个项目通过将抽象的数学概念（位置+朝向编码编码）与具体的物理模型（魔方）相结合，为孩子们提供了一种直观理解建模过程的方式。通过观察数据、动作与模型之间的对应关系，孩子们可以更好地理解如何将现实世界的问题转化为计算机可以处理的数据结构。

## 后续扩展计划（3阶及更高阶）

### 可行性结论

当前框架可以扩展到 3 阶，理论上也可继续扩展到更高阶；但不建议直接在现有 `Cube2x2` 代码上“硬改参数”。

原因是当前实现围绕 2 阶角块模型设计（8 个角块、3 面贴纸、2 阶专用状态编码），而 3 阶会新增棱块与中心块，状态编码与可视化规则都不同。

### 难度评估

- 扩展到 **3 阶**：中高难度（建议分阶段迭代）
- 扩展到 **N 阶通用**：高难度（需先抽象数据模型与旋转框架）

### 风险控制原则（避免影响当前2阶）

为了不影响当前稳定的 2 阶版本，建议采用“并行新增，不覆盖旧逻辑”的方式：

1. 保留现有：
   - `Cube2x2.gd`
   - `CubePiece.gd`
   - `scenes/main.tscn`
2. 新增扩展模块：
   - `CubeNxN.gd`（参数化阶数）
   - `Cubelet.gd`（通用小块，支持 1/2/3 贴纸）
   - `scenes/main_3x3.tscn`（独立场景）
3. 通过独立入口测试 3 阶，确认稳定后再考虑合并 UI 与数据接口

### 推荐分阶段实施

#### 阶段1：先做可视化与层旋转（低风险）

- 支持 3 阶生成与显示（26 个可见小块）
- 支持 U/D/L/R/F/B 基本层旋转
- 目标：模型形态、颜色、转动视觉正确

#### 阶段2：再做3阶状态表示（教学版）

- 先做可读状态输出（如数组/字符串）
- 再决定是否实现完整的“置换+朝向”编码

#### 阶段3：抽象到N阶

- 参数化层选择与旋转
- 增加中间层/宽层支持
- 逐步统一输入、显示与状态接口

  - 状态编码格式

这样可以确保文档与代码持续一致，便于后续教学与维护。

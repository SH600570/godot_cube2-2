# 二阶魔方教学工具

## 项目概述

这是一个使用 Godot 引擎开发的二阶魔方教学工具，旨在通过可视化和数据建模的方式，帮助孩子们理解魔方的工作原理和建模概念。

## 工程架构

项目采用了清晰的三层架构：

1. **CubePiece**（角块基类）
   - 负责：显示、颜色、自身状态
   - 不负责：旋转逻辑

2. **Cube2x2**（魔方总控制器）
   - 负责：创建所有块、管理旋转、层转动、坐标更新
   - 旋转逻辑全部写在这里

3. **UIController**（输入控制器）
   - 负责：处理用户输入、显示魔方状态

## 功能特点

### 1. 8位3进制数表示魔方状态

魔方的状态通过一个8位的3进制数来表示，每一位对应一个角块的状态。具体规则如下：

- 按照固定顺序检查每个角块的**逻辑坐标**（`piece_positions` 数组中约定的 8 个位置，坐标为 `(+/-0.5, +/-0.5, +/-0.5)`）
- 每个角块的状态由其**当前朝上的贴纸颜色**（世界坐标系的 Y+ 方向）决定
- 颜色到值的映射：白色=0，红色=1，绿色=2，蓝色=0，黄色=1，橙色=2

### 2. 实时状态显示

界面上会实时显示当前魔方的状态，以8位3进制数的形式呈现。每次旋转后，状态会自动更新。

### 3. 教学模式

#### 数据、动作与模型的对应关系

- **数据**：8位3进制数表示的魔方状态
- **动作**：用户通过键盘输入进行的旋转操作
- **模型**：可视化的3D魔方模型

#### 教学步骤建议

1. **基础概念介绍**：
   - 介绍魔方的基本结构（8个角块）
   - 解释3进制数的概念
   - 说明如何用3进制数表示魔方状态

2. **数据与模型对应**：
   - 展示初始状态下的3进制数
   - 指出每个数字对应的角块
   - 解释颜色如何映射到数字

3. **动作与数据对应**：
   - 执行一次旋转操作
   - 观察状态数字的变化
   - 分析哪些数字发生了变化，为什么

4. **模型与动作对应**：
   - 观察旋转时魔方的视觉变化
   - 解释旋转轴和方向
   - 理解层旋转的概念

5. **综合练习**：
   - 让孩子尝试不同的旋转组合
   - 观察状态数字的变化
   - 尝试根据状态数字预测魔方的外观

## 操作说明

- **U**：上层顺时针旋转
- **D**：下层顺时针旋转
- **L**：左层顺时针旋转
- **R**：右层顺时针旋转
- **F**：前层顺时针旋转
- **B**：后层顺时针旋转
- **空格**：重置魔方

## 技术实现

- **CubePiece.gd**：角块基类，管理颜色和显示  
  每个角块是一个 `Node3D`，包含 1 个立方体网格（`CubeBody`）和 6 个平面贴纸网格（`Face_X+/-`、`Face_Y+/-`、`Face_Z+/-`）。  
  代码中只在“外露的 3 个方向”上给贴纸上色（例如 `(0.5,0.5,0.5)` 这个角块只在 +X、+Y、+Z 上有颜色），其余 3 个面始终朝向魔方内部并被隐藏，因此整个魔方一共只有 3×8=24 个彩色面，符合真实二阶魔方的 24 张贴纸。

- **Cube2x2.gd**：魔方总控制器，实现旋转逻辑和状态计算  
  内部使用：
  - `piece_positions`：长度为 8 的数组，约定 8 个角块的逻辑坐标顺序，坐标为 `(+/-0.5, +/-0.5, +/-0.5)`，这样 8 个角块的中心点刚好拼成一个 2×2×2 的正方体；
  - `pieces`：场景中所有角块节点的列表；
  - `pos_to_piece`：从逻辑坐标到角块实例的字典，便于 O(1) 查找。

  旋转时，控制器会：
  - 先根据逻辑坐标筛选出某一层的 4 个角块（例如右层 `x>0`）；
  - 临时创建一个 `Node3D` 作为旋转枢轴，把这 4 个角块作为子节点挂到枢轴下面；
  - 对枢轴做 90° 旋转，实现“整层刚体旋转”；
  - 再把 4 个角块挂回原来的父节点，根据它们当前的位置重新计算逻辑坐标（吸附到最近的 `+/-0.5`），并按新的逻辑坐标重建 `pos_to_piece` 映射；
  - 最后让每个角块重新隐藏内部 3 个面。

  状态计算时，控制器会：
  - 按 `piece_positions` 的顺序依次取出每个逻辑坐标对应的角块；
  - 查询这个角块“当前朝上的贴纸颜色”（世界坐标系的 Y+ 方向）；
  - 利用颜色到数字的映射表，把 8 个角块的朝上颜色转成一个 8 位 3 进制字符串。

- **UIController.gd**：输入处理和状态显示  
  负责把键盘输入（U/D/L/R/F/B/空格）转换成对 `Cube2x2` 的方法调用，并在左上角的 `Label` 上实时显示当前的 8 位 3 进制状态。

## 设计思路与架构优化

### 设计原则

项目遵循以下软件工程原则：

1. **单一职责原则**：每个模块只负责一项核心功能
2. **职责分离**：明确划分各模块的职责，避免逻辑混乱
3. **封装性**：隐藏内部实现细节，提供清晰的接口
4. **可扩展性**：设计灵活，便于后续功能扩展
5. **性能优化**：针对关键操作进行性能改进

### 架构决策

1. **旋转逻辑集中管理**：所有旋转逻辑都集中在 `Cube2x2` 控制器中，而不是分散在各个角块中，这样可以更好地管理层旋转的复杂性

2. **数据结构设计**：
   - 使用 `pos_to_piece` 字典实现逻辑坐标到块实例的快速映射
   - 采用固定的 `piece_positions` 数组定义角块顺序，确保状态计算的一致性

3. **状态计算优化**：
   - 通过映射快速查找块，将状态计算的时间复杂度从 O(n²) 降低到 O(n)
   - 只在旋转操作后更新状态，避免不必要的计算

4. **代码组织**：
   - 清晰的变量和方法命名
   - 详细的注释说明
   - 模块化的代码结构

### 优化措施

1. **性能优化**：
   - 添加 `pos_to_piece` 映射，提高块查找效率
   - 移除不必要的 `_process` 方法，减少性能消耗

2. **代码质量优化**：
   - 移除重复代码，使用已定义的常量和数组
   - 保持代码风格一致，提高可读性
   - 添加详细注释，提高可维护性

3. **可维护性优化**：
   - 确保映射一致性，在旋转和重置操作后正确更新 `pos_to_piece` 映射
   - 保持职责分离，确保各模块功能清晰
   - 遵循面向对象原则，保持良好的封装性

4. **可扩展性优化**：
   - 模块化的设计便于添加新功能
   - 清晰的接口设计便于与其他系统集成
   - 灵活的数据结构便于后续扩展

## 扩展建议

1. 添加更详细的状态解释，显示每个数字对应的具体角块和颜色
2. 实现步骤记录功能，让孩子可以看到一系列动作如何改变状态
3. 添加简单的解谜挑战，给定目标状态，让孩子通过旋转达到目标
4. 可视化3进制数到颜色的映射过程，帮助孩子理解编码原理

## 总结

这个项目通过将抽象的数学概念（3进制数）与具体的物理模型（魔方）相结合，为孩子们提供了一种直观理解建模过程的方式。通过观察数据、动作与模型之间的对应关系，孩子们可以更好地理解如何将现实世界的问题转化为计算机可以处理的数据结构。

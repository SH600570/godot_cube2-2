# 二阶魔方教学工具

## 项目概述

这是一个使用 Godot 引擎开发的二阶魔方教学工具，旨在通过可视化和数据建模的方式，帮助孩子们理解魔方的工作原理和建模概念。

## 工程架构

项目采用了清晰的三层架构：

1. **CubePiece**（角块基类）
   - 负责：显示、颜色、自身状态
   - 不负责：旋转逻辑

2. **Cube2x2**（魔方总控制器）
   - 负责：创建所有块、管理旋转、层转动、坐标更新
   - 旋转逻辑全部写在这里

3. **UIController**（输入控制器）
   - 负责：处理用户输入、显示魔方状态、部分 UI（如重置确认对话框）

4. **OrbitCamera**（`scripts/OrbitCamera.gd`，挂在 `Camera3D` 上）
   - 负责：调试/教学用轨道相机（右键旋转视角、滚轮缩放、中键平移目标）

## 当前已实现（体验与表现）

- **层旋转**：`Cube2x2.rotate_*` 使用临时 pivot 做整层 90° 旋转，逻辑坐标吸附 ±0.5 后重建 `pos_to_piece`。
- **状态输出**：`get_cube_state()` 按角块固定 `piece_id` 顺序返回长度为 8 的整型数组（位置 3 位 + 朝向 2 位打包为十进制数，与上文编码示例一致）。
- **显示**：`PIECE_SPACING`（约 1.03）产生块间细缝；`DISPLAY_SCALE` / `DISPLAY_OFFSET` 控制整体大小与构图；前向轴约定为 **+Z**（`DIR_FRONT`）。
- **渲染**：深色环境 + 多盏补光 + 贴纸材质增强（乳白、饱和度等，见 `CubePiece.gd`）。
- **相机**：轨道操作见上；**层转动**为 **Tween 插值**（约 0.2s）+ 动画中 `is_rotating` 锁输入。
- **面外光环**：每个外侧面 **仅一套** 双 90° 弧（半径约 **0.7**、环粗 **0.2**、斜向 45° 后 **[0°,90°]** 与 **[180°,270°]**），两段末端箭头 **反向**，分别表示顺转/逆转；每帧用该层四块在 **6** 张贴纸中 **与层外法线最一致** 的面求平均面心/法线，避免转动后仍用固定 `local_face` 导致指示陷入魔方内部；左键 **`Cube2x2._input` → 物理射线** 命中分段碰撞体后触发 `rotate_*`（见 `FaceHintArrows.gd`）。
- **音频**：**未实现**。

### 2. 实时状态显示

界面上会实时显示当前魔方的状态，以数组（8个数字）的形式呈现。每次旋转后，状态会自动更新。

### 3. 教学模式

#### 数据、动作与模型的对应关系

- **数据**：数组（8个数字）表示的魔方状态
- **动作**：用户通过键盘输入进行的旋转操作
- **模型**：可视化的3D魔方模型

#### 教学步骤建议

1. **基础概念介绍**：
   - 介绍魔方的基本结构（8个角块）
   - 解释空间位置的概念
   - 解释朝向编码的概念
   - 打印编码表，包含8个位置的读取顺序。8个角在不同位置颜色对应的朝向编码。

2. **数据与模型对应**：
   - 展示初始状态下的数组（8个数字）
   - 解读数组中每个数字的含义，与实物对应。
   - 解释颜色如何映射到数字

3. **动作与数据对应**：
   - 执行一次旋转操作
   - 观察状态数字的变化
   - 分析哪些数字发生了变化，为什么

4. **模型与动作对应**：
   - 观察旋转时魔方的视觉变化
   - 解释旋转轴和方向
   - 理解层旋转的概念

5. **综合练习**：
   - 让孩子尝试不同的旋转组合
   - 观察状态数字的变化
   - 尝试根据状态数字预测魔方的外观

## 操作说明

- **视角**：鼠标 **右键拖拽** 绕魔方旋转相机；**滚轮** 缩放；**中键拖拽** 平移观察中心（`OrbitCamera`）。
- **推荐**：在魔方 **六个外侧面** 上点击 **半透明弯箭头**（青/橙各一枚）执行该层 **顺时针 / 逆时针** 转动，与当前视角无关；右侧 **U/D/L…** 文字按钮已隐藏。
- **U**：上层顺时针旋转，shift+U为逆时针旋转
- **D**：下层顺时针旋转，shift+D为逆时针旋转
- **L**：左层顺时针旋转，shift+L为逆时针旋转
- **R**：右层顺时针旋转，shift+R为逆时针旋转
- **F**：前层顺时针旋转，shift+F为逆时针旋转
- **B**：后层顺时针旋转，shift+B为逆时针旋转
- **空格**：重置魔方到初始状态，需要输入"reset"确认，防止误操作。

## 技术实现

- **CubePiece.gd**：角块基类，管理颜色和显示  
  每个角块是一个 `Node3D`，包含 1 个立方体网格（`CubeBody`）和 6 个平面贴纸网格（`Face_X+/-`、`Face_Y+/-`、`Face_Z+/-`）。  [朝向定义]呼应关系 Face_X+对应R,Face_X-对应L,Face_Y+对应U,Face_Y-对应D,Face_Z+对应F,Face_Z-对应B。
  代码中只在“外露的 3 个方向”上给贴纸上色,可以参考[8个位置读取顺序]里面的方向，那些方向是能看到的面，其他3个面是隐藏的。

- **Cube2x2.gd**：魔方总控制器，实现旋转逻辑和状态计算  
  内部使用：
  - `piece_positions`：8 个角块的逻辑坐标（±0.5），顺序与 README 位置编号 0–7 一致；
  - `pieces`：所有角块节点引用；
  - `pos_to_piece`：`Vector3` 逻辑坐标 → `CubePiece` 实例，便于 O(1) 查找；
  - 每个角块带有固定 `piece_id`（0–7），`get_cube_state()` 按 id 排序后输出 8 个编码值。

  旋转时：按层选块 → 临时 `RotatePivot` 父节点 → **Tween 插值**绕世界轴转 90° → 子节点挂回 → 吸附逻辑坐标 → 重建 `pos_to_piece` → 刷新各块外露/内部面。

  状态计算：对每个角块根据当前逻辑位置查 `orientation_mapping`，结合世界方向读贴纸颜色，得到朝向码，与位置索引组合为十进制数组元素（详见上文「综合后的初始状态编码」与 U 示例）。

- **UIController.gd**：输入与界面  
  将键盘（含 Shift 逆时针）映射到 `Cube2x2` 的 `rotate_*`；左上角 `Label` 显示 `str(cube.get_cube_state())`；空格触发带 `reset` 确认的重置；右侧 **U/D/L…** 文字旋转按钮默认隐藏（主操作改为面外箭头）。

- **FaceHintArrows.gd**：六面外 **单套** 双弧光环 + 反向箭头  
  每块在 6 个贴纸中选取与层外法线最一致的一面再对四层平均，得到面心与 **n**；`StaticBody3D` 扁盒分两段弧放置；由 **`Cube2x2._input`** 调用 `_try_pick_rotate`；动画期间受 `is_rotating` 约束。

## 设计思路与架构优化

### 设计原则

项目遵循以下软件工程原则：

1. **单一职责原则**：每个模块只负责一项核心功能
2. **职责分离**：明确划分各模块的职责，避免逻辑混乱
3. **封装性**：隐藏内部实现细节，提供清晰的接口
4. **可扩展性**：设计灵活，便于后续功能扩展
5. **性能优化**：针对关键操作进行性能改进

### 架构决策

1. **旋转逻辑集中管理**：所有旋转逻辑都集中在 `Cube2x2` 控制器中，而不是分散在各个角块中，这样可以更好地管理层旋转的复杂性

2. **数据结构设计**：
   - 使用 `pos_to_piece` 字典实现逻辑坐标到块实例的快速映射
   - 采用固定的 `piece_positions` 数组定义角块顺序，确保状态计算的一致性

3. **状态计算优化**：
   - 通过映射快速查找块，将状态计算的时间复杂度从 O(n²) 降低到 O(n)
   - 只在旋转操作后更新状态，避免不必要的计算

4. **代码组织**：
   - 清晰的变量和方法命名
   - 详细的注释说明
   - 模块化的代码结构

### 优化措施

1. **性能优化**：
   - 添加 `pos_to_piece` 映射，提高块查找效率
   - 移除不必要的 `_process` 方法，减少性能消耗

2. **代码质量优化**：
   - 移除重复代码，使用已定义的常量和数组
   - 保持代码风格一致，提高可读性
   - 添加详细注释，提高可维护性

3. **可维护性优化**：
   - 确保映射一致性，在旋转和重置操作后正确更新 `pos_to_piece` 映射
   - 保持职责分离，确保各模块功能清晰
   - 遵循面向对象原则，保持良好的封装性

4. **可扩展性优化**：
   - 模块化的设计便于添加新功能
   - 清晰的接口设计便于与其他系统集成
   - 灵活的数据结构便于后续扩展

## 总结

这个项目通过将抽象的数学概念（位置+朝向编码编码）与具体的物理模型（魔方）相结合，为孩子们提供了一种直观理解建模过程的方式。通过观察数据、动作与模型之间的对应关系，孩子们可以更好地理解如何将现实世界的问题转化为计算机可以处理的数据结构。

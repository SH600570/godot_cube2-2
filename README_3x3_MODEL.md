# 三阶魔方教学建模文档（草案）

> 目标：先把“数据模型”定义清楚，再做代码实现。  
> 你可以在此文档基础上人工逐条确认，再进入开发。

## 1. 项目目标与范围

本文档定义三阶魔方（3x3x3）的教学建模规则，强调：

1. **模型可视化**：3D 中正确显示 26 个可见块（8角 + 12棱 + 6中心）
2. **动作一致性**：U/D/L/R/F/B 及其逆时针动作的层旋转正确
3. **数据可读性**：状态可编码、可打印、可验证
4. **教学友好**：动作前后能观察“模型变化 <-> 状态变化”

本草案先定义“教学版状态编码”，不追求一次到位实现竞速魔方求解器格式。

---

## 2. 坐标系与方向定义（统一约定）

为了避免 Godot 内置方向歧义，统一约定：

- 右：`+X`（R）
- 左：`-X`（L）
- 上：`+Y`（U）
- 下：`-Y`（D）
- 前：`+Z`（F）
- 后：`-Z`（B）

> 注：在代码里建议显式定义：
> - `DIR_RIGHT = Vector3(1,0,0)`
> - `DIR_UP = Vector3(0,1,0)`
> - `DIR_FRONT = Vector3(0,0,1)`
> - 其余方向取相反向量，避免直接混用 `Vector3.FORWARD/BACK`。

---

## 3. 三阶块类型与数量

三阶总可见块：26（不含核心）

1. **角块 Corner**：8 个，每块 3 个可见贴纸
2. **棱块 Edge**：12 个，每块 2 个可见贴纸
3. **中心块 Center**：6 个，每块 1 个可见贴纸

可见贴纸总数：`8*3 + 12*2 + 6*1 = 54`

---

## 4. 逻辑坐标建模（推荐）

采用离散逻辑坐标：`x,y,z ∈ {-1,0,1}`

- 角块：`|x|+|y|+|z| = 3`
- 棱块：`|x|+|y|+|z| = 2`
- 中心块：`|x|+|y|+|z| = 1`
- 核心（不显示）：`(0,0,0)`

### 4.1 位置到世界空间

建议显示间距参数：

- `PIECE_SPACING`：相邻小块中心间距（例如 `1.03`）
- 世界位置：`world_pos = logic_pos * PIECE_SPACING`

---

## 5. 初始配色定义（教学版）

采用你当前项目一致的配色语义：

- U（上，+Y）= 白 `white`
- D（下，-Y）= 黄 `yellow`
- F（前，+Z）= 红 `red`
- B（后，-Z）= 橙 `orange`
- L（左，-X）= 绿 `green`
- R（右，+X）= 蓝 `blue`

每个小块只给其**外露方向**着色：

- 角块：3 面着色
- 棱块：2 面着色
- 中心块：1 面着色

---

## 6. 三阶位置编号（建议）

为了教学和编码清晰，建议“分组编号”而不是直接 0~25 混排：

1. 角块编号：`C0..C7`
2. 棱块编号：`E0..E11`
3. 中心块编号：`M0..M5`

### 6.1 角块编号（建议沿用二阶角顺序）

- `C0 = URF` ( +X,+Y,+Z )
- `C1 = UFL` ( -X,+Y,+Z )
- `C2 = ULB` ( -X,+Y,-Z )
- `C3 = UBR` ( +X,+Y,-Z )
- `C4 = DRB` ( +X,-Y,-Z )
- `C5 = DBL` ( -X,-Y,-Z )
- `C6 = DLF` ( -X,-Y,+Z )
- `C7 = DFR` ( +X,-Y,+Z )

### 6.2 棱块编号（建议）

- `E0 = UR`  (+X,+Y,0)
- `E1 = UF`  (0,+Y,+Z)
- `E2 = UL`  (-X,+Y,0)
- `E3 = UB`  (0,+Y,-Z)
- `E4 = FR`  (+X,0,+Z)
- `E5 = FL`  (-X,0,+Z)
- `E6 = BL`  (-X,0,-Z)
- `E7 = BR`  (+X,0,-Z)
- `E8 = DR`  (+X,-Y,0)
- `E9 = DF`  (0,-Y,+Z)
- `E10 = DL` (-X,-Y,0)
- `E11 = DB` (0,-Y,-Z)

### 6.3 中心块编号（建议）

- `M0 = U` (0,+Y,0)
- `M1 = D` (0,-Y,0)
- `M2 = F` (0,0,+Z)
- `M3 = B` (0,0,-Z)
- `M4 = L` (-X,0,0)
- `M5 = R` (+X,0,0)

---

## 7. 旋转动作定义（面向观察者）

基础动作：`U D L R F B`  
逆时针动作：`U' D' L' R' F' B'`

建议约定：  
“某面顺时针”定义为**从该面正对观察**时看到的顺时针。

实现上可统一为：

- 选层（4角 + 4棱 + 1中心，共9块）
- 以该层中心创建 pivot
- 旋转 ±90°
- 回写逻辑坐标（吸附到 -1/0/1）

---

## 8. 状态编码（教学版，推荐先做）

先用可读、可验证的编码，便于教学，不必一开始就上最压缩格式。

## 8.1 编码目标

输出一个对象（或字典）：

- `corners`: 8项
- `edges`: 12项
- `centers`: 6项（可选，三阶中心方向固定，可只做颜色一致性校验）

### 8.2 角块编码（位置+朝向）

每个角块输出：

- `pos`: 当前在哪个角位置（0~7）
- `ori`: 朝向编码（0~2）

可组合为一个数字（示例）：

- `corner_code = (pos << 2) | ori`

### 8.3 棱块编码（位置+朝向）

每个棱块输出：

- `pos`: 当前在哪个棱位置（0~11）
- `ori`: 朝向编码（0~1）

可组合为：

- `edge_code = (pos << 1) | ori`

### 8.4 中心块编码

教学版可先固定中心，不编码朝向，仅校验颜色正确。

## 9. 可视化与材质建议（延续当前项目经验）

1. 角块本体用较深灰，增强贴纸对比
2. 白色贴纸使用乳白并加轻微发光
3. 可配置“内部面调试模式”：
   - 正常：内部面隐藏
   - 调试：内部面染紫，便于排错
4. 使用多光源补光（主光 + 顶光 + 侧后补光）
5. 相机使用轨道控制（右键旋转、滚轮缩放）

---

## 10. 开发落地建议（低风险）

为了不影响现有 2 阶稳定版本，建议并行新增：

1. 新脚本：`CubeNxN.gd`（先只支持 size=3）
2. 新脚本：`Cubelet.gd`（通用小块）
3. 新场景：`main_3x3.tscn`
4. 复用现有：`OrbitCamera.gd`、`UIController` 的输入框架（适配新接口）

## 11. 验收清单（人工可核对）

### 11.1 模型验收

- [ ] 初始显示 26 个可见小块
- [ ] 六面颜色分布正确（每面 9 贴纸）
- [ ] 小块间有细缝，视觉像真实魔方

### 11.2 动作验收

- [ ] U/D/L/R/F/B 正逆时针动作都正确
- [ ] 任意连续操作后，模型不“开花”
- [ ] 连续执行动作再逆动作可回到初始状态

### 11.3 状态验收

- [ ] 每次动作后状态对象同步更新
- [ ] 角块位置/朝向编码稳定
- [ ] 棱块位置/朝向编码稳定

---

## 12. 预留讨论项（你可人工确认）

1. 角块朝向编码顺序（0/1/2 的定义）是否沿用当前二阶规则
2. 棱块朝向编码（0/1）的颜色读取顺序
3. 是否需要加入中间层动作（M/E/S）与宽层动作
4. 是否在教学版中显示“可读文字状态 + 压缩编码”双输出

---

## 13. 最小可行实现（MVP）

建议第一版只做：

1. 3阶可视化 + 基本 6 面旋转
2. 角块编码（先不做棱块朝向）
3. UI 显示状态与重置

待稳定后再补齐棱块编码与更高阶抽象。
